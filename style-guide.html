01 — Overview

Design Character

Both apps share an immediately recognizable visual language: compact, card-based widgets centered on the screen, saturated single-hue palettes per app, soft-square buttons with heavy drop shadows, and a consistent grid of icon-driven controls. The aesthetic is playful but purposeful — closer to physical hardware (a physical timer or retro calculator) than to a flat dashboard.

Despite being built years apart, the structural decisions are almost identical: same card width (~275–280px), same button size (45×45px), same border-radius logic (2% on cards, 15% on rectangular buttons, 50% on circular ones), and the same shadow system across all interactive elements.

Each app uses a monochromatic background tinted to its hue. The Calculator establishes the fullest version of this rule — its page background (#bae5e5) is a light desaturated teal directly derived from the card color. The Pomodoro's background is a neutral grey (rgba(230,230,230,.96)), not themed. The To-do follows the Calculator's pattern with a soft sage wash (rgba(220,235,210,.96)) derived from the lime card hue.

The intended pattern (established by the Calculator, followed by To-do) is: page background = very light, low-saturation tint of the card's dominant hue. This keeps the card feeling native to the page rather than dropped onto a neutral canvas. Opacity near 0.96 is also consistent — a small transparency that softens the color slightly.

⚠ Border-radius on wide elements

Percentage-based border-radius behaves differently on non-square elements. On a square button, 15% gives a subtle soft corner. On a wide rectangle (like an input or display screen), 15% creates an extreme pill/oval shape. The source code uses 3% for wide display elements — this is the correct value to use.

The shadow system conveys a full tactile hierarchy: cards float above the page with a colored shadow, buttons rest with a medium diffuse shadow, hover tightens the shadow (button feels closer), active press adds a deep inset shadow (button feels depressed). The calculator display uses an inset shadow to look like a sunken screen.

07 — Design Principles

Recurring Patterns

🎨

Monochromatic Theming

Each app picks one saturated hue and uses tints/shades of it for every element. Theme color never mixes with the other app's hue.

🟦

Semantic Color Constants

Destructive (red #b35764) and confirm (green #20b286) override the theme color regardless of which app. Meaning > decoration.

📦

Fixed-Width Widget

Both apps are ~280px wide, vertically centered. The card is a self-contained object — not a responsive layout, but a discrete tool.

🔲

Soft-Square Buttons

15% border-radius strikes between a circle and a rectangle. Feels touchable, not sterile. Consistent 45×45px across both apps.

💡

Tactile Shadow Hierarchy

Cards glow with colored shadow. Buttons press inward on :active. Display screen sits inset. Every state has a physical metaphor.

🏋️

Bold, White-on-Color Text

All button text is white, font-weight 600. High contrast against saturated backgrounds. No neutral grey in interactive elements.
